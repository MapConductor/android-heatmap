### `HeatmapPointState`

A stateful class representing a single data point within a heatmap layer.

This class holds the geographical position, weight, and optional metadata for a heatmap point. It is designed for use with Jetpack Compose, as its properties are mutable states that trigger recomposition upon change.

#### Signature

```kotlin
class HeatmapPointState(
    position: GeoPointInterface,
    weight: Double = 1.0,
    id: String? = null,
    extra: Serializable? = null,
) : ComponentState
```

#### Parameters

| Parameter  | Type                  | Default | Description                                                                                                                                                           |
| :--------- | :-------------------- | :------ | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `position` | `GeoPointInterface`   | -       | **Required.** The geographical location of the heatmap point.                                                                                                         |
| `weight`   | `Double`              | `1.0`   | The intensity or weight of the point. Higher values contribute more to the heatmap's intensity in that area.                                                          |
| `id`       | `String?`             | `null`  | An optional unique identifier for the point. If not provided, a stable ID is automatically generated based on the `position` and `extra` data.                         |
| `extra`    | `Serializable?`       | `null`  | Optional, serializable data that can be attached to the point. This can be useful for storing additional information related to the point, such as metadata or a model ID. |

### Methods

#### `fingerPrint()`

Generates a compact, hash-based representation of the point's current state. This is useful for efficient change detection.

-   **Signature:** `fun fingerPrint(): HeatmapPointFingerPrint`
-   **Returns:** `HeatmapPointFingerPrint` - An object containing the hash codes of the point's properties.

#### `asFlow()`

Creates a reactive stream (`Flow`) that emits a new `HeatmapPointFingerPrint` whenever any property of the `HeatmapPointState` changes. This allows for observing the state of a point programmatically. The flow is configured to only emit when the state has genuinely changed.

-   **Signature:** `fun asFlow(): Flow<HeatmapPointFingerPrint>`
-   **Returns:** `Flow<HeatmapPointFingerPrint>` - A Kotlin Flow that emits a new fingerprint on each state change.

### Example

The following example demonstrates how to create and update a `HeatmapPointState`.

```kotlin
import com.mapconductor.heatmap.HeatmapPointState
import com.mapconductor.core.features.GeoPointInterface
import kotlinx.coroutines.launch
import kotlinx.coroutines.runBlocking
import java.io.Serializable

// A mock implementation of GeoPointInterface for the example.
data class GeoPoint(val latitude: Double, val longitude: Double) : GeoPointInterface {
    override fun getLatitude(): Double = latitude
    override fun getLongitude(): Double = longitude
}

// A mock implementation for extra data.
data class PointMetadata(val name: String, val category: String) : Serializable

fun main() = runBlocking {
    // 1. Create a GeoPoint for the heatmap point's position.
    val initialPosition = GeoPoint(latitude = 40.7128, longitude = -74.0060)
    
    // 2. Create an instance of HeatmapPointState.
    val heatmapPoint = HeatmapPointState(
        position = initialPosition,
        weight = 5.0,
        extra = PointMetadata("Statue of Liberty", "Landmark")
    )

    println("Initial Point ID: ${heatmapPoint.id}")
    println("Initial Weight: ${heatmapPoint.weight}")
    println("Initial Extra Data: ${heatmapPoint.extra}")

    // 3. Observe changes to the point's state using asFlow().
    val job = launch {
        heatmapPoint.asFlow().collect { fingerprint ->
            println("Point state changed! New fingerprint: $fingerprint")
            println("Current weight: ${heatmapPoint.weight}")
        }
    }

    // 4. Update the point's weight. This will trigger the flow collector.
    println("\nUpdating weight to 10.0...")
    heatmapPoint.weight = 10.0
    
    // Give the collector a moment to process the change
    kotlinx.coroutines.delay(100)

    // 5. Update the position.
    println("\nUpdating position...")
    heatmapPoint.position = GeoPoint(latitude = 40.7130, longitude = -74.0062)

    // Give the collector a moment to process the change
    kotlinx.coroutines.delay(100)

    job.cancel() // Clean up the coroutine
}
```

### `HeatmapPointFingerPrint`

A data class that represents a snapshot of a `HeatmapPointState`'s state using the hash codes of its properties.

Its primary purpose is for efficient change detection, particularly as the emission type for the `asFlow()` stream in `HeatmapPointState`. By comparing fingerprints, the system can quickly determine if a point's state has changed without needing to compare each property individually.

#### Signature

```kotlin
data class HeatmapPointFingerPrint(
    val id: Int,
    val position: Int,
    val weight: Int,
    val extra: Int,
)
```

| `position` | `Int`  | The hash code of the `HeatmapPointState`'s `position` property.          |
| `weight`   | `Int`  | The hash code of the `HeatmapPointState`'s `weight` property.            |
| `extra`    | `Int`  | The hash code of the `HeatmapPointState`'s `extra` property.             |
